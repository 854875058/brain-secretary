# 工作流/闭环

自动沉淀来自 QQ / 桥接层的长期记忆，按时间追加。


## 自动打回闭环（2026-03-09）

- 用户不希望被动承担可恢复的环境问题；像缺依赖、缺测试工具、项目未克隆、默认分支未确认，这些应先由内部 agent 自己处理。
- 推荐闭环：大脑拆单 -> 技术号实施/补环境/跑测试 -> 验收号独立复核 -> 发现问题则大脑带着明确 blocker 自动打回技术号 -> 再验收 -> 最后统一汇报。
- 只有在需要用户独有信息时才询问用户，例如：授权口令、验证码、业务取舍、私有仓额外权限、最终偏好裁决。
- 如果用户已经给过一次仓库地址或项目别名，应把它沉淀到项目注册表和长期记忆，后续不要再反复追问。
